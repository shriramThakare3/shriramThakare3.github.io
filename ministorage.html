Mini Cloud Storage API
1.0.0
Google Drive-style file storage backend. Upload files via multipart/form-data, organise into virtual folders, search by name, rename, and download by ID. Files live on disk; metadata in PostgreSQL.
Base URL: http://localhost/ · Run locally with Docker Compose · docker compose up --build
files — upload, download, manage▼
POST /files/upload Upload a file
Parameters
NameTypeDescription
folder_idstring(optional) Target folder UUID. Omit for root.
Request Body
// multipart/form-data file: <binary> // required — max 100 MB
Responses
201File metadata: id, filename, size, content_type, folder_id, created_at.
400Empty file.
404Folder not found.
413File exceeds 100 MB limit.
GET /files List files
Parameters
NameTypeDescription
searchstringCase-insensitive filename substring match
folder_idstringFilter by folder. Use 'none' for root-level only.
pageintegerPage (default: 1)
limitintegerPer page (default: 20, max: 100)
Responses
200{ total, page, limit, files: [...] }
GET /files/{id} Download a file
Parameters
NameTypeDescription
id*stringFile UUID
Responses
200Binary file stream. Content-Disposition: attachment.
404Not found.
410Record exists but file missing from disk.
DELETE /files/{id} Delete a file
Parameters
NameTypeDescription
id*stringFile UUID
Responses
204DB record deleted and bytes removed from disk.
404Not found.
PATCH /files/{id}/rename Rename a file
Parameters
NameTypeDescription
id*stringFile UUID
Request Body
{ "filename": "new-name.pdf" }
Responses
200Updated file object.
404Not found.

POST /folders Create a folder
Request Body
{ "name": "Documents", "parent_id": "uuid..." // optional — omit for top-level }
Responses
201Folder object with id, name, parent_id.
404Parent folder not found.
GET /folders List folders
Parameters
NameTypeDescription
parent_idstringFilter by parent. Omit = top-level only. 'all' = every folder.
Responses
200Array of folder objects.
