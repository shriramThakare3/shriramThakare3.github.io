Authentication Server API
1.0.0
JWT authentication backend. Register, login, and refresh tokens. Access tokens are short-lived (30 min); refresh tokens are long-lived (7 days), stored as SHA-256 hashes. Supports RBAC, token rotation with reuse detection, and per-session logout.
Base URL: http://localhost/ · Run locally with Docker Compose · docker compose up --build
auth — register, login, token management▼
POST /auth/register Register a new user
Request Body
{ "username": "alice", // 3-64 chars, [a-zA-Z0-9_-] "email": "alice@example.com", "password": "secret123" // min 8 chars }
Responses
201{ access_token, refresh_token, token_type: bearer, expires_in: 1800 }
409Username or email already registered.
422Validation error.
POST /auth/login Login with credentials
Request Body
{ "username": "alice", // accepts username OR email "password": "secret123" }
Responses
200{ access_token, refresh_token, token_type, expires_in }
401Invalid username or password. (Intentionally identical for wrong user vs wrong password — prevents enumeration.)
403Account is disabled.
POST /auth/refresh Get new access token
Request Body
{ "refresh_token": "eyJ..." }
Responses
200{ access_token, token_type, expires_in }
401Token invalid, expired, revoked, or reuse detected. If reuse detected, entire token family is revoked.
POST /auth/logout Logout — revoke refresh token
Request Body
{ "refresh_token": "eyJ..." }
Responses
200{ "message": "Logged out successfully." }
POST /auth/logout-all Logout all devices
⚠ Requires Authorization: Bearer <access_token> header.
Responses
200{ "message": "Logged out from N session(s)." }
401Invalid or expired access token.
users — profile & admin▼
GET /users/me Get own profile
⚠ Requires Authorization: Bearer <access_token> header.
Responses
200{ id, username, email, role, is_active, created_at, updated_at }
401Token missing or invalid.
PATCH /users/me Update own profile
⚠ Requires Authorization: Bearer <access_token> header.
Request Body
{ "username": "new_name", // optional "email": "new@email.com" // optional }
Responses
200Updated profile.
409Username or email already taken.
DELETE /users/me Deactivate own account
⚠ Requires Authorization: Bearer <access_token> header. Soft-delete — sets is_active=false.
Responses
204Account deactivated.
GET /users List all users (admin only)
⚠ Requires admin role. Add Authorization: Bearer <admin_access_token> header.
Parameters
NameTypeDescription
searchstringSearch username or email
pageintegerPage
limitintegerPer page (max 100)
Responses
200{ total, page, limit, users: [...] }
403Admin privileges required.
